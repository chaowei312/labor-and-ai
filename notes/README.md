# Notes

Planning, design, and process documents for the DSAN 5200 narrative submission. None of these files are deliverables in themselves — the deliverable is the rendered site under [`../narrative_site/`](../narrative_site/) and the data + figure pipeline under [`../scripts/`](../scripts/) and [`../data/`](../data/). These notes exist so the editorial reasoning and audit trail are reproducible.

| File | Purpose |
|------|---------|
| [`PROJECT_PLAN.md`](PROJECT_PLAN.md) | Editorial framing, sector pillars (services × manufacturing equal weight), risks, time window. |
| [`VISUALIZATION_PLAN.md`](VISUALIZATION_PLAN.md) | Design vocabulary, Reuters/Pudding-style scroll plan, locked typography/color spec. |
| [`CLAUDE_DESIGN_PROMPT.md`](CLAUDE_DESIGN_PROMPT.md) | The single brief used to drive the Claude Design pass that produced the design system. Quoted in the [appendix AI usage log](../narrative_site/appendix.qmd). |

For graders: the audit trail you most likely want is `CLAUDE_DESIGN_PROMPT.md` (AI-assistance disclosure) alongside the rendered site itself.
